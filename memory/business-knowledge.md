## Jobber API & MCP Integration — Research Summary (updated 2026-03-28)

### CORRECTION — No simple API token in Jobber Settings
There is NO "Settings > API Access" option for generating a simple API token in Jobber's main app.
This is a common misconception repeated in some community MCP READMEs (including BusyBee3333).
The ONLY path to API credentials is via the Jobber Developer Center (developer.getjobber.com).
Source: developer.getjobber.com/docs/getting_started/ and docs/custom_integrations/ (verified 2026-03-28)

### Plan restrictions — CONFIRMED
- API access is available on ALL Jobber plans (Core, Connect, Grow, Plus)
- There is NO plan gate on using the Developer Center or OAuth API
- The only plan-related upsell is "API Tour with API Access" on Plus plan ($99 value) = guided onboarding session, not a feature gate
- Source: getjobber.com/pricing/ plan comparison table (verified 2026-03-28)

### 1. Official Jobber Public API — YES, confirmed
- GraphQL API at `https://api.getjobber.com/api/graphql`
- Auth: OAuth 2.0 ONLY (no simple API token option exists)
- Credentials come from: developer.getjobber.com → create an App → get Client ID + Client Secret
- Rate limits: 2,500 requests per 5 minutes; 10,000 query cost per request (GraphQL complexity)
- Playground: developer.getjobber.com → your app → Test in GraphiQL
- Data accessible: clients, jobs, quotes, invoices, visits, requests, timesheets, expenses, products, line items, forms, taxes, properties, team

### 2. How to get API credentials — exact steps (verified from docs)
1. Go to developer.getjobber.com (separate from your main Jobber account login)
2. Create a Developer Center account (free, separate login)
3. Click "NEW" to create an app
4. Fill in: App name, Developer name, App description (required); OAuth Callback URL (optional for self-use)
5. Select Scopes (what data the app can access — e.g. clients, jobs, invoices)
6. App is created in "Draft" state — you get a Client ID and Client Secret
7. For self-use (Monarch only): use "Test in GraphiQL" to complete OAuth and get a 60-minute access token
8. For persistent use: implement the OAuth 2.0 Authorization Code flow to get access_token + refresh_token
9. The refresh token can be used to get new access tokens indefinitely (until disconnected)

### 3. Custom Integrations (self-use — no App Marketplace needed)
- You can keep app in "Draft" state — no need to publish to App Marketplace
- Draft apps can connect to up to 5 paying Jobber accounts without approval
- For Monarch (single account), Draft state is sufficient forever
- Source: developer.getjobber.com/docs/custom_integrations/ (verified 2026-03-28)

### 4. OAuth 2.0 flow summary
- Authorization URL: `https://api.getjobber.com/api/oauth/authorize?response_type=code&client_id=CLIENT_ID&redirect_uri=CALLBACK_URL&state=STATE`
- Token endpoint: POST `https://api.getjobber.com/api/oauth/token`
- Access token expiry: 60 minutes (use refresh token to get new ones)
- Refresh token: long-lived (rotates optionally); persists until app disconnected or plan downgraded
